## 二、铁律（不可违背）

1. **不准编造**：没实跑就写"未实测"，不写"已验证"。所有核心数值必须来自真实代码/数据/推导，可复算。可分解结果写"近似最优解"，不写"全局最优"。
2. **算法服帖题目，不加模戏**：每用一个算法说清——为什么适合本题、输入输出、规模限制、如何验证。禁止为显高级乱堆神经网络/遗传/退火/粒子群/TOPSIS/灰色预测。
3. **闭环必须完整**：问题分析 → 假设 → 符号 → 模型 → 算法 → 结果 → 检验 → 优缺点。不只给公式不解释变量；不只给结果不解释合理性；不只写算法名不给求解流程。
4. **语言像投稿论文，不像 AI 作文**：少用"具有重要意义""效果显著""充分体现"。多用"本文首先…然后…最后…""由表可知…""结果表明…""这说明…"。

## 三、算法分层策略

- **递进结构**：基础模型 → 加约束 → 扩规模 → 多目标综合。例：TSP → 带时间窗 VRP → 大规模可分解 → 多车容量 VRP。
- **小规模**：精确算法（DP/枚举剪枝/整数规划/精确求解器），作为后续算法的正确性基线。
- **大规模**：可分解/混合算法。必须说明：为何精确不可行、为何完整 QUBO 不可行、初始解怎么来、局部如何搜索、稳定性如何判断。
- **创新点要真实**：来自抽象更整齐、变量更简洁、约束更干净、分层更自然、验证更充分。禁用"首创""国际领先"。
- **可复现**：每个算法交代输入、决策变量、目标函数、约束、关键参数（初始温度/降温系数/最大迭代/邻域操作/接受准则/随机种子）、终止条件、输出、代码位置。

## 四、写作语句模板

- **问题分析**：交代题目本质 + 与上一问的扩展关系。
- **模型建立**："设…为决策变量。以…为目标函数。考虑…约束，建立如下条件。最终得到…模型。"
- **算法选择**："由于…规模较小，采用…精确算法；规模扩大到…时直接求解会…，故采用…混合算法。优势在…，不足在…，后续通过…检验。"
- **结果分析**：每个关键数值后加 2–4 句——说明什么、是否符合直觉、与约束什么关系、对实际什么意义、有无异常。
- **结论**：逐题对应——"针对问题一，得到…；针对问题二，得到…；…综合来看，本文模型说明…"
- **摘要**：每问的方法 + 核心结果 + 关键数值 + 模型优势 + 是否检验，全文浓缩，不写背景。

## 五、必做检验

参数扰动 ±5/10/20%；不同算法对比；资源/权重变化；误差与可行性检查；稳定性与复杂度分析。**没有灵敏度分析的国奖论文几乎不存在。**

## 六、图表纪律

每张图表必须有明确作用且被正文引用。禁止无解释/无编号/无单位/无图例/截图模糊/公式乱码。优先三线表，公式居中编号，变量首次出现必释义。

## 七、修改论文 Checklist（每次改完跑一遍）

1. 题目要求是否全部回答（路线/总时间/每客户开始时间/早晚到/总惩罚/总目标都给齐）
2. 摘要含每问方法+结果+关键数值
3. 公式变量是否解释、结果数值与代码输出一致
4. 图表编号连续且正文引用
5. 算法说明输入/输出/参数/流程
6. 是否有"虚构运行/虚构真机/虚构成功"
7. 结论与最终结果一致
8. 附录代码与正文方法一致
