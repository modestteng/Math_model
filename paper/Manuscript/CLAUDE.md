说明：CIM 真机不是装饰，而是用于真实量子计算环境下求解 QUBO；对于超过 550 比特的问题，采用分段 subQUBO 或 block-diag 策略；真机返回结果必须解码并重新评价完整业务目标。

|---|---:|---:|---|---:|---:|---|---|

写作重点：

## 7. 真机表现方式必须克制但突出

不要写：

> 量子真机显著优于所有经典算法。

除非有严格数据支持。推荐写：

> 本文将 CIM 真机作为 QUBO 子问题求解器，用于验证所构造 QUBO 矩阵在真实量子计算环境下的可求解性。对于大规模问题，本文并不直接构造全局 QUBO，而是将真机用于局部路径重排、分段排序或 block-diag 合并子问题，并在解码后回到原始业务目标进行统一评价。

如果真机结果比经典算法好，可以写：

> 在该实例中，CIM 返回解经解码后取得了更低的综合目标值，说明局部 subQUBO 优化能够改善经典启发式初始解。

如果真机结果不如经典算法，也可以写：

> 虽然该次 CIM 返回解未优于最佳经典启发式结果，但其满足 QUBO 约束并能够解码为可行路径，验证了本文 QUBO 编码与真机调用流程的可行性。

这样更真实，评委更信。

## 10. LaTeX 编译与清理规则（强制 · 写完即编译）

**强制约定**：每次写完或修改任何 `.tex` 文件后，**必须立刻自动编译**，编译成功后**必须立刻清理中间产物**，整个动作不能等用户提醒。流程如下：

1. **编译**（在该 `.tex` 所在目录执行）：

```bash
xelatex -interaction=nonstopmode -halt-on-error <name>.tex
```

或对包含目录/参考文献的主文件用 `latexmk -xelatex <name>.tex`。

2. **失败必须立刻修**：如果 `xelatex: Command for 'xelatex' gave return code 1` 或 stdout 报 `Undefined control sequence` / `Missing $ inserted` 等，**当场修改 `.tex` 源文件**（补宏包、修语法），重跑直到生成 `.pdf` 为止；**不允许把失败留给用户**。常见缺包：`\mathbb` 缺 `amssymb`、`\includegraphics` 缺 `graphicx`、`\toprule` 缺 `booktabs`、`\begin{tikzpicture}` 缺 `tikz` + `pgfplots`。

3. **编译成功后立刻清理**（强制，不留中间产物）：

```bash
rm -f <name>.aux <name>.log <name>.out <name>.toc <name>.synctex.gz \
      <name>.fdb_latexmk <name>.fls <name>.blg <name>.bbl <name>.bcf \
      <name>.run.xml <name>.xdv
```

PowerShell 等价：`Remove-Item <name>.{aux,log,out,toc,synctex.gz,fdb_latexmk,fls,blg,bbl,bcf,run.xml,xdv} -ErrorAction SilentlyContinue`。

4. **验证只剩两类文件**：编译目录下只允许保留 `*.tex` 和 `*.pdf`；如有 `refs.bib` 等必要源文件一并保留。多余 `.aux/.log/.out/.toc/.synctex.gz/.fdb_latexmk/.fls/.blg/.bbl/.bcf/.run.xml/.xdv` 全部删除。

5. **报告给用户**：编译完成后用一句话告知 PDF 路径与页数（如``已编译为 `Abstract.pdf`，3 页''），不需要让用户再提醒``跑一下编译''。

**只保留**：源文件 `.tex`、最终论文 `.pdf`、必要的 `.bib`、项目原始材料、图片、表格、代码、结果文件。
